# 📏 Measure

**Color:** yellow · **Emoji:** 📏

> Source note: in the original PRCSS* template this phase is labeled "Mesure" (French spelling). We standardize to **Measure** in English outputs but preserve the source spelling when reproducing the template verbatim.

## Intent
After launch, track whether the design moved the metrics that justified the project. Close the loop with Discover and Define.

## Typical methods to add to this phase
- KPI dashboarding (Amplitude, Mixpanel, GA, custom)
- Cohort analysis
- Post-launch user interviews
- Ongoing CSAT / NPS / SUS
- Funnel / retention monitoring
- Support ticket & bug trend review
- Retrospective with the team

## Typical outputs
- A post-launch scorecard against the `📈 The Metrics` block from Project Info.
- A list of follow-up actions (added back into the database as new actions in earlier phases — this is the "loop" in the framework).
- A written retrospective.

## Exit criteria
You have evidence of impact (or lack of impact) against the Objectives set in Define, and the team has agreed on the next iteration — or on closing the project.
