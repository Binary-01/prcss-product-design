# 📦 Deliver

**Color:** green · **Emoji:** 📦

## Intent
Ship the design. Ensure engineering, content, and QA have everything they need, then hand off cleanly.

## Typical methods to add to this phase
- Developer handoff (specs, tokens, prototypes)
- Design QA on staging / pre-prod
- Release notes / changelog
- Documentation in the design system / wiki
- Stakeholder sign-off
- Launch checklist

## Typical outputs
- A shipped feature.
- Handoff doc (see `design:design-handoff` skill).
- Updated design system.
- A brief for the Measure phase.

## Exit criteria
Engineers can implement without ambiguity, the feature is live (or accepted into the release train), and you've defined what "success" looks like for Measure.
